🎯 HTML vs Markdown 수학 표기법 렌더링 비교

📝 블로그 포스팅 미리보기

제약 조건:
  • -231 ≤ x ≤ 231 - 1

✨ HTML 방식 (제안된 방식)

저장 형태:
<code>-2<sup>31</sup> &le; x &le; 2<sup>31</sup> - 1</code>
브라우저 렌더링:
-231 ≤ x ≤ 231 - 1
✅ 장점:
• 수학적으로 정확한 상첨자 표현
• 모든 브라우저에서 일관된 렌더링
• 전문적이고 깔끔한 외관
• CSS 스타일링 가능
• LeetCode 원본과 동일한 형태
❌ 단점:
• 데이터베이스에서 읽기 복잡
• 텍스트 검색 어려움
• 저장공간 약간 증가

📝 Markdown 방식 (현재 방식)

저장 형태:
-2^31^ <= x <= 2^31^ - 1
브라우저 렌더링:
-2^31^ ≤ x ≤ 2^31^ - 1
✅ 장점:
• 데이터베이스에서 읽기 쉬움
• 텍스트 검색 용이
• 개발자 친화적
• 저장공간 효율적
❌ 단점:
• 수학 표기법 시각적 한계
• 전문성 부족
• 블로그 파서 의존성
• 플랫폼별 렌더링 차이

🔍 실제 사용 예시 비교

HTML 방식 (추천):

제약 조건:

  • 1 ≤ nums.length ≤ 104
  • -231 ≤ nums[i] ≤ 231 - 1
  • -231 ≤ target ≤ 231 - 1
Markdown 방식 (현재):

제약 조건:

  • 1 ≤ nums.length ≤ 10^4^
  • -2^31^ ≤ nums[i] ≤ 2^31^ - 1
  • -2^31^ ≤ target ≤ 2^31^ - 1

💡 권장사항

HTML 방식을 추천하는 이유:

  1. 블로그 최적화: 이 프로젝트의 최종 목표가 블로그 포스팅
  2. 시각적 품질: 수학 표기법의 전문적이고 정확한 표현
  3. 사용자 경험: 블로그 독자들에게 더 나은 가독성 제공
  4. 호환성: 모든 주요 브라우저와 블로그 플랫폼에서 지원
